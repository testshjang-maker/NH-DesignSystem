Usage rules

Principles

When building new screens, default to these behaviours unless a spec explicitly overrides.

✓ Do
  • Primary CTAs → Filled · Purple, size 40 by default.
  • Destructive CTAs → Filled · Warning. Always pair with a Border · Black 취소 button.
  • Text inside cards → GRAY_8. Titles → GRAY_9. Supporting → GRAY_7.
  • Use Body size 14 as the default body size for all UI.
  • Form field heights: 40 in cards & rows, 48 in modals & onboarding.
  • Tag sizes: SM for inline status, XS only inside dense tables, LG for feature highlights.
✕ Don't
  • Don't invent new hex values — pick from Color.ts.
  • Don't use multiple purples on one screen. PURPLE_6 is the only brand accent.
  • Don't stack filled buttons. One primary per view.
  • Don't use Borderless black for destructive actions — use Borderless warning.
  • Don't set font-size directly. Use Body / Heading components.
  • Don't use the Gradient tokens outside of marketing/upsell surfaces.
